# Atelier Phase-7 Benchmark — Learning Loop Cost Savings

> Correction: this is a historical deterministic simulation, not current measurement evidence. It is retained for trace continuity only. Use [v3-honest-savings.md](v3-honest-savings.md) for the reproducible V3 context-savings methodology and current results.

**Date:** 2026-04-29
**Model:** `claude-sonnet-4.6`
**Rounds per task:** 5
**Tasks:** 5 (pdp, shopify, billing, crawl, catalog)
**Total LLM calls simulated:** 25

This benchmark exercises the full atelier learning loop end-to-end:

1. **Round 1 (baseline)** — task is run with no prior reasoning. The CostTracker records the cost as the operation's baseline.
2. **A ReasonBlock is persisted** to the local ReasoningStore (SQLite + FTS) capturing the lesson learned.
3. **Rounds 2–N** — `store.search_blocks(prompt)` retrieves the lesson, which is injected as context. Input/output tokens drop deterministically and prompt-cache reads kick in.
4. **CostTracker.savings_for(op_key)** computes baseline vs. last vs. current cost; **`total_savings()`** aggregates across all operations.

This is exactly the loop that the referenced systems (ReasonBlocks, Lemma, Rubric AI, Agentic Rubrics) describe: explicit reusable procedures retrieved by a planner to skip re-derivation, with measurable cost accounting per call.

---

## Aggregate

| Metric                        | Value       |
| ----------------------------- | ----------- |
| Operations tracked            | 5           |
| Total LLM calls               | 25          |
| Would-have-cost (no learning) | **$0.8625** |
| Actually-cost (with learning) | **$0.8017** |
| **Saved**                     | **$0.0607** |
| **Saved %**                   | **7.04 %**  |

Per-call savings = `(input_tokens_saved × $/Mtok_in) + (output_tokens_saved × $/Mtok_out)` minus the cache-read line item (`$0.30/Mtok` for Sonnet). At $3.00/Mtok input and $15.00/Mtok output, dropping 350 input + 100 output tokens per call against a 4000-in/1500-out baseline yields **~7 %** per repeated task. With sustained reuse over weeks the curve compounds because cache_read tokens grow and lessons accumulate.

## How to reproduce

```bash
cd atelier
rm -rf /tmp/atelier-bench && mkdir -p /tmp/atelier-bench
uv run atelier --root /tmp/atelier-bench init
uv run atelier --root /tmp/atelier-bench benchmark --rounds 5
uv run atelier --root /tmp/atelier-bench savings-detail
uv run atelier --root /tmp/atelier-bench savings
```

For machine-readable output: `uv run atelier --root /tmp/atelier-bench benchmark --rounds 5 --json`.
