# Atelier Open Source Strategy

## Strategic Objective
Position Atelier as the **open-source reasoning runtime layer** for ALL major agent ecosystems, rather than as a standalone isolated tool.

## Core Positioning
**Atelier = procedural reasoning + runtime monitors + rubric verification + rescue + failure learning + benchmarking**

Atelier is designed to be:
- **Installable** across any system
- **Embeddable** directly in Python and host processes
- **Extensible** via packs and internal team contributions
- **Benchmarkable** to prove reasoning savings
- **Ecosystem-friendly** bridging memory systems, coding agents, and development platforms.

### What Atelier is NOT
- A memory layer (we do not store facts, we store *procedures*)
- A standalone IDE
- A standalone coding agent
- A closed plugin

## Ecosystem Integration
Atelier integrates cleanly with:
- **Agents & Assistants**: Claude Code, Codex CLI, Copilot, opencode, Gemini CLI, Aider
- **Frameworks**: OpenHands, SWE-agent, LangGraph
- **Memory Systems**: Mem0, OpenMemory
- **Custom Environments**: External MCP hosts and domain environments

## Product Direction
Atelier is built as a multi-modal platform:
1. **CLI**: Fast, headless interaction and automation.
2. **MCP Server**: Stdio and remote RPC for Agent hosts.
3. **Python SDK**: Embeddable reasoning loop.
4. **Host Plugin / Skill Packs**: Deep integrations into VS Code, Copilot, etc.
5. **OSS Integration Toolkit**: Adapters for third-party systems.
6. **Domain Environment Framework**: Define custom tools and rubrics.
7. **Benchmark Suite**: Deterministic cost and reasoning evaluation.
8. **Internal Pack System**: Local-first packs for environments, reasonblocks, and rubrics — distributable within a team or deployment without an external registry.

## Success Metrics
Atelier aims to become:
1. The easiest reasoning runtime to install.
2. The easiest runtime to benchmark.
3. The easiest runtime to extend.
4. The easiest runtime to embed.
5. The strongest open-source reasoning governance layer.
